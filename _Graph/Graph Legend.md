# 🗺️ Graph Legend

This note explains how the Obsidian graph works for this study vault.

## Tag Structure

Each video note is tagged along three axes so the graph clusters meaningfully:

| Tag pattern | What it groups |
|-------------|---------------|
| `#week/1` … `#week/4` | Notes from the same week |
| `#topic/naming` | Notes about naming |
| `#topic/functions` | Notes about functions |
| `#topic/structure` | Notes about code structure & form |
| `#topic/tdd` | Notes about test-driven development |
| `#topic/architecture` | Notes about high-level design |
| `#topic/claude-code` | Notes from the Anthropic course |
| `#source/clean-code` | Uncle Bob video series notes |
| `#source/anthropic` | Anthropic Claude Code course notes |
| `#status/watched` | Completed |
| `#status/todo` | Not yet started |

## How to Read the Graph

- **Dense clusters** = deeply connected topic areas (good!)
- **Isolated nodes** = a note with no links yet — go add connections
- **Hub nodes** = `Weekly Study Plan` and the week index notes act as anchors

## Tips for a Beautiful Graph

1. Always fill in the **Connections** section of each note — even one link makes a visible difference.
2. Use `[[note name]]` wikilinks generously.
3. After finishing all 4 weeks, filter by `#topic/tdd` to see how the TDD notes link to everything else.
